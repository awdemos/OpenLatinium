# Three Coexisting Compilation Paths

OpenLatinum maintains three distinct compilation paths: an original single-pass syntax-directed path, an AST-based path with explicit semantic analysis, and an IR-based path with optimizations. This decision was made to support teaching compiler construction incrementally — each path represents a progressively more sophisticated compiler architecture — while also serving as a testbed for experimentation. The trade-off is significant maintenance burden: the paths share no common infrastructure (different parsers, different semantic systems, different code generators), and only the AST path is currently functional for the full test suite. A future consolidation into a single well-tested path would require choosing one architecture and porting features from the others.
